git相关命令

git的三个工作区域

  1. 工作目录
    .git所在的目录，用来存储新创建的文件和修改的文件。
  2. 暂存区
    用于存储半成品。
  3. 本地仓库
    存储整个项目的快照，保存当前暂存区中的一份代码。
    一般用于存储成品，成品最终会上线。

git文件的四个状态

  1. 未跟踪
    新创建的文件的状态
  2. 已暂存
    位于暂存区的文件的状态
  3. 已提交
    位于本地仓库的文件的状态
  4. 已修改
    暂存区的文件又被修改了时的状态

常见命令

  • git config 配置本地仓库
    git config –global user.name 配置用户名
    git config –global user.email 配置用户邮箱
  • git config –list 查看用户信息
  • git init 初始化本地仓库
    git init –bare 初始化远程仓库
  • git status 查看当前仓库的状态
  • git add 文件名 将工作目录中的文件添加到暂存区
    git add * 或-A 表示将所有文件添加到暂存区
  • git commit -m ‘备注信息’ 将暂存区的文件添加到本地仓库
  • git log 查看本地仓库的提交历史(包括commit id)

    回滚操作

  • git rm –cached 文件列表 删除暂存区指定的文件

  • git reset 回滚快照
    git reset –hard 回滚本地仓库的快照，并使用快照覆盖暂存区和工作目录
    git reset –soft 只回滚本地仓库的快照
    git reset –mixed （默认）回滚本地仓库的快照和暂存区，但不覆盖工作目录
    特殊用法：
    回滚最后一次提交（回到倒数第二次提交）
    git reset –hard HEAD^ 或 git reset –hard 倒数第二次的commit id

  • git checkout 文件名 回滚暂存区已经修改的内容（暂存区修改后发现修改的不对时使用）
    git checkout . 表示回滚暂存区所有文件的所有修改，谨慎使用。

分支操作

  • git branch 查看分支
  • git branch 分支名称 创建分支
  • git checkout 分支名称 切换分支
  • git checkout -b 分支名称 创建并切换到分支
  • git merge 分支名称 合并分支
  • git branch -d 分支名称 删除分支（分支必须合并过之后才能被删除）
  • git branch -D 分支名称 强制删除分支

远程仓库的操作

  • git init –bare 初始化远程仓库
  • git clone 远程仓库地址 项目名称（可选参数） 克隆远程仓库的项目
  • git push 远程仓库地址 本地分支名称 上传代码到远程仓库
  • git pull 远程仓库地址 远程分支名称 下载代码到本地分支
