清除浮动的方式

语法： 选择器{clear:both;}

清除方式

  • 额外标签法(不常使用)
    在浮动元素末尾添加一个空标签：
  • 父级元素添加overflow属性
    给父级元素添加overflow:hidden/auto/scroll;三种都可实现。
  • after伪元素清除浮动
    语法：
    .clearfix:after{ content:".",
                 display:block,
                 height:0,
                 visibility:hidden,
                 clear:both;
                }
.clearfix { *zoom:1;}  ie6、7专有
  • 空余字符法
    语法：
    .clearfix:after{ content:"\200B",
                 display:block,
                 height:0,
                 clear:both;
                }
.clearfix { *zoom:1;}  ie6、7专有
  • 双伪元素法
    语法：
    .clearfix:after,.clearfix:after{ content:" ",
                                 display:table,
                                }
.clearfix:after{ clear:both;}
.clearfix { *zoom:1;}  ie6、7专有

外边距合并(外边距塌陷)

相邻块元素垂直外边距的合并

当上下相邻的两个块元素相遇时，如果上面的元素有下外边距，下面的元素有上外边距，则它们的距离并不是两者之和，而是两者中的较大者。
解决方案：避免即可。

嵌套块元素垂直外边距的合并

对于两个嵌套关系的块元素，如果父元素没有上内边距及边框，则父元素的上外边距会与子元素的上外边距发生合并，合并后的外边距为两者中的较大者，即使父元素的上外边距为零也会发生合并。
解决方案：1. 为父元素定义1px的上边框或上内边距；
          2. 为父元素添加overflow:hidden;
          3. 给父元素设置padding,而不要给子元素设上外边距。

盒子边框 border

语法 border:border-width border-style border-color;

border-style 取值：
    - none 没有边框(默认)
    - solid 单实线
    - dashed 虚线
    - dotted 点线
    - double 双实线
注意：border-style 为必须设置的属性，否则默认为none,显示不了没有边框

关于边框的初始化

1. 表单的边框清零
原因：因为每种浏览器的表单默认边框颜色不同，样式不同，因此需要将默认的清零，自己定义样式和颜色，从而达到浏览器兼容的效果。
语法：input{ border:0; }
2. 表格的边框
语法：table{ border-collapse:collapse;border-spacing:0; }
    border-collapse:collapse; 表示表格的边框合并在一起
    border-spacing:0;  表示边框之间的间距为零
3. 轮廓线outline
原因：由于链接获得焦点后有虚线、文本框或者文本域获得焦点时，谷歌等浏览器会有蓝色边框，为了浏览器兼容故将它们的轮廓线取消。
语法：input{outline:none;}  a:focus{outline:none;}
